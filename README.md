## 🗂 GitHub 제출 가이드

### 1️⃣ 폴더 구조 생성 (필수)
레포지토리 최상단에 **본인 이름(또는 GitHub ID)** 폴더를 생성하고, 그 하위에 **주차별 폴더**를 생성하여 파일을 관리합니다.

```text
repo-root/
 └─ 채주혁/
     ├─ week1/
     │   ├─ baekjoon_10828.java
     │   └─ programmers_완주하지못한선수.java
     ├─ week2/
     └─ week3/
```
### 2️⃣ 파일명 규칙 (필수)
파일명에는 공백이 없어야 하며, 아래 형식을 준수합니다.

- 백준: baekjoon_문제번호.확장자 (예: baekjoon_10828.java)

- 프로그래머스: programmers_문제이름.확장자 (예: programmers_완주하지못한선수.py)

### 3️⃣ Pull Request (PR) 생성 방법
1. Fork: 원본 레포지토리를 본인 계정으로 Fork합니다.

2. Clone: Fork한 레포지토리를 로컬에 Clone 합니다.

3. Commit & Push: 문제 풀이 파일을 폴더 구조에 맞춰 추가한 뒤 본인 원격 레포지토리에 Push합니다.

4. PR 생성: 본인 레포지토리의 Pull requests 탭에서 원본 레포지토리의 main 브랜치로 PR을 보내어 Merge 합니다. PR은 매일 올리지 않고, 해당 주차에 푼 문제들(최소 5문제)을 모아서 하나의 PR로 보냅니다

5. PR 제목 규칙: [Week N] 이름 풀이 제출 (예: [Week1] 채주혁 풀이 제출)
